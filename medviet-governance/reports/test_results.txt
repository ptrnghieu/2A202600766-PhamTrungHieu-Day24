Bugs found & fixed during this verification pass
--------------------------------------------------
1. src/pii/detector.py — VN_CCCD and VN_PHONE PatternRecognizer instances
   were missing supported_language="vi". They defaulted to "en", but the
   AnalyzerEngine was restricted to supported_languages=["vi"], so any
   analyze(language="vi", entities=["VN_CCCD"/"VN_PHONE"]) call raised
   ValueError: No matching recognizers were found to serve the request.
   Fixed by adding supported_language="vi" to both recognizers.

2. tests/test_pii.py, src/api/main.py, src/quality/validation.py —
   pd.read_csv() on patients_raw.csv was not given an explicit dtype for
   the cccd / so_dien_thoai / patient_id columns. Pandas inferred these
   numeric-looking columns as int64, silently stripping leading zeros.
   Vietnamese phone numbers always start with "0", so so_dien_thoai was
   corrupted on every row (always read as a 9-digit number, defeating
   the VN_PHONE regex \b0[35789]\d{8}\b which requires the leading 0);
   cccd was corrupted on any value containing leading zeros. Fixed by
   passing dtype={"cccd": str, "so_dien_thoai": str, "patient_id": str}
   to every read_csv() call against this file. Confirmed fix: phone
   detection went from 0% to 100%, cccd detection from 82% to 100%.
